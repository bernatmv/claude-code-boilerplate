# Phase 6 â€” Auth End-to-End Implementation Plan

**Goal:** Wire email+password auth on web + mobile, with middleware-protected routes on web and a session-aware layout on mobile. Google OAuth is **stubbed** behind documented-but-unused buttons; the shared auth infrastructure is ready to accept it when downstream apps supply Google credentials.

**Architecture:**

- **Web.** `@supabase/ssr` creates a browser client for client components and a server client (cookie-bound) for server components, route handlers, and server actions. Four auth routes under `/[locale]/(auth)/{sign-in,sign-up,forgot-password,reset-password}` call server actions that wrap `supabase.auth.signInWithPassword`, `signUp`, `resetPasswordForEmail`. A `/[locale]/(protected)` route group wraps `profile`, `settings`, `dashboard` â€” its layout redirects unauthenticated users to `/sign-in`. Middleware additionally refreshes the cookie on every request.
- **Mobile.** `@repo/api-client` exposes a new factory `createSupabaseMobileClient(url, anonKey, storage)` that plugs an `expo-secure-store`-backed `SupportedStorage` adapter into the Supabase client. Mobile adds sign-in / sign-up / forgot screens under `(auth)/` group. Root layout watches `onAuthStateChange` and redirects to `(tabs)` if signed in or `(auth)/sign-in` if not.
- **Shared.** `useSession(client)` already exists in `@repo/api-client`. Add a `signOut(client)` helper and an `AuthProvider` that publishes the current session via React context for consumers that don't want TanStack Query wiring.

**Tech Stack:** `@supabase/ssr ^0.5`, existing `@supabase/supabase-js ^2.45`, `expo-secure-store ^14`, React Hook Form ^7, `@hookform/resolvers ^3`, existing Zod schemas from `@repo/validation`.
